OS CRUSH API

oscrush.js

OS Crusher implements a really fun and addictive game. It is very similiar to the popular Candy Crush Saga. There is a seperate module that implements the OS Crusher functionality. The OS Crusher module can be accessed through a global variable named game. Example: game.initialize() will initialize the game.

Inside this global variable we have the following functions available:

Function Name Parameters Return Value Description
initialize None None This function initializes the grid
addListener callback: provides a callback function to be called
type of listener: provides the type of listener		 None This function adds a listener and provides a callback function that will be called.
List of Listeners:
"drawGrid": to represent the grid as images of icons

"updateABox": to update elements position in the view according to the change made in the grid by the model

"drawNewBox": to draw new icons created

"deleted": to update view when the model deletes elements

"pioneer": to provide the pioneer destructive effect listener, blast all the same kind

"troy": to provide the troy destructive effect listener, tornado

"bouclier":to provide the shield listener, when we want to avoid user from moving element around while the game is busy , it avoids error.
"soundListenner" to provide the sounds effects
checkGrid boolean true boolean This function checks the grid to see if there are any matches. It will return true when matches are found or false otherwise. It does not change or delete anything from the grid.
checkGrid none or boolean false boolean This function checks the grid to see if there are any matches. It will return true when matches find or false otherwise. It does change the grid by deleting the matches.
checkVerticals elt: provides the identification number of the element we are searching
row: is the row to check
column: is the column where the element is.		 An object with a count of the element, its row, and its column This function looks for elements that are repeating more than 2 times in a row/column. It will return an object with a count of how many times the element is repeating, its row, and its column.
checkHorizontals elt: provides the identification number of the element we are searching matches
row: is the row to check
column: is the column where the element is.		 An object with a count of the element, its row, and its column This function looks for any elements that are repeating more than 2 times in a row from an element to its right side. It returns an object with a count of how many time the element is repeating, its row, and its column.
destroyGridIndex top: provides the row
left: provides the column		 None This function can be called by the view to delete a specific index from a row inside the grid
getGridIndex id: provides the identification of the element. None This function returns the id of a specific location in the grid
findLocationOf id: provides the identification of the element. False or an object containing the positions, row, and column. This function can find the position of the element pass to it if they exist on the board
deleteDone ids: provides the identification of the element removed. None This is a function that will be called when the view is done deleting elements
tryMoveClick ids: provides the identification of the element None This function will check if a movement is legal and updates the view
tryMoveDrag ids: provides the identification of the element None This function checks if a movement is legal after a drag, and updates the view
deleteAll type: provides the type of the element. For example 0 is for android None This function deletes all of the types of a specific element from the game
deleteAll None None This function deletes all of the elements from the game
